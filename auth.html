Authentication is identity. Authorization is permissions. Mixing them makes everyone sad.

Recommendations

  • Use a well-supported identity provider (OIDC/OAuth2).
  • Use short-lived tokens, rotate keys, and validate audiences/issuers.
  • Separate user auth from service-to-service auth.
  • For sessions: HttpOnly + Secure cookies; avoid storing tokens in localStorage.

Common foot-guns

  • Missing token revocation strategy.
  • Over-scoped tokens (“admin” everywhere).
  • Not checking tenant/org boundaries.
