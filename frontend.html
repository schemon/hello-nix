Frontend is UX + performance + reliability.

Basics

  • Ship less JavaScript.
  • Handle loading, empty states, and errors explicitly.
  • Accessibility: semantic HTML first, then ARIA if needed.
  • Keep design tokens (spacing, color, typography) consistent.

Observability

  • Track key user flows. Measure time-to-interactive.
  • Capture client errors with stack traces and release versions.
