Backends should be boring, predictable, and cheap to operate.

API design

  • Version your API. Document breaking changes.
  • Idempotency for writes (especially payments/webhooks).
  • Explicit timeouts and retries (with jitter).

Data

  • Use migrations. Avoid manual prod edits.
  • Index for queries you actually run.
  • Have a clear retention policy.
